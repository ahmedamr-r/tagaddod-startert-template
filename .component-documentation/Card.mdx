# Card Component

The Card component is a versatile container that serves as a flexible building block for any content. It provides optional header and footer sections while maintaining a consistent visual design with equal padding on all sides.

### Default Equal Padding Behavior

### RTL Best Practices

1. **Always use ThemeProvider** for proper RTL support and font switching
2. **Choose directional icons carefully** - arrows should point in the reading direction

The Card component works seamlessly with other design system components:

- **Buttons**: For actions in headers and footers
- **Badges**: For status indicators and tags
- **Avatars**: For user representation
- **Icons**: For visual hierarchy and actions
- **Select/Input**: For embedded forms
- **Switch/Checkbox**: For settings toggles

## Accessibility

- **Keyboard Navigation**: Interactive and clickable cards are focusable with Tab
- **Screen Reader Support**: Proper semantic structure with header/body/footer
- **ARIA Labels**: When clickable, cards include appropriate ARIA attributes
- **High Contrast**: All text meets WCAG contrast requirements
- **Reduced Motion**: Respects `prefers-reduced-motion` for animations
- **Focus Indicators**: Clear focus rings for keyboard navigation

## Performance Considerations

- **React.memo**: Component is optimized to prevent unnecessary re-renders
- **CSS Modules**: Efficient styling with scoped CSS classes
- **Minimal DOM**: Clean HTML structure without wrapper elements
- **Optimized Animations**: Hardware-accelerated transforms for smooth performance
- **Bundle Size**: Lightweight implementation with tree-shaking support
