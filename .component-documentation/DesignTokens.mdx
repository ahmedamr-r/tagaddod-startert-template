# Design Tokens Reference

This document provides a comprehensive reference for all Tagaddod design tokens available in the design system. **Always use these tokens instead of hardcoded values** to maintain consistency.

## Quick Reference

All tokens are available as CSS custom properties from `@tagaddod-design/react/dist/styles/styles.css`:

```css
/* Import in your CSS */
@import '@tagaddod-design/react/dist/styles/styles.css';

/* Use in CSS */
.my-component {
  color: var(--t-color-text-primary);
  padding: var(--t-space-400);
  font: var(--t-typography-body-md);
}
```

```tsx
// Use in React components with style prop
<div style={{ 
  color: 'var(--t-color-text-primary)',
  padding: 'var(--t-space-400)',
  font: 'var(--t-typography-body-md)'
}}>
```

## ‚ö†Ô∏è Best Practices

### DO ‚úÖ
- Always use design tokens instead of hardcoded values
- Reference this document when creating custom components
- Use complete typography tokens (`--t-typography-*`) for consistent text styling
- Follow the spacing scale for consistent layouts
- Use semantic color tokens (`--t-color-text-primary`) over palette tokens

### DON'T ‚ùå
- Use arbitrary values like `color: #333` or `margin: 16px`
- Mix design system tokens with hardcoded values
- Create custom spacing or color values outside the system
- Ignore the typography scale and use random font sizes

### Token Import
```css
/* Always import the complete design system */
@import '@tagaddod-design/react/dist/styles/styles.css';
```
