# Logo

The Logo component displays the Tagaddod brand logo with various size and color options. It supports both static display and clickable functionality.

## Features

- **Multiple sizes**: Small, medium, large, and custom variants
- **Custom dimensions**: Specify exact width and/or height with automatic aspect ratio preservation
- **Color variants**: Primary, white, and inherit options
- **Clickable**: Can be rendered as a button with onClick handler
- **Accessibility**: Proper ARIA labels and keyboard navigation
- **RTL Support**: Works correctly in right-to-left layouts
- **Responsive**: Scales appropriately across different screen sizes
- **Aspect ratio preservation**: Automatically maintains logo proportions when using custom sizes

## Usage

```tsx
import { Logo } from '@tagaddod-design/react';

// Basic usage
<Logo />

// With predefined size and color
<Logo size="large" color="white" />

// Custom size with both dimensions
<Logo size="custom" width={200} height={50} />

// Custom size with only width (height auto-calculated)
<Logo size="custom" width={300} />

// Custom size with only height (width auto-calculated)
<Logo size="custom" height={60} />

// Custom size with string units
<Logo size="custom" width="15rem" height="3rem" />

// Clickable logo
<Logo clickable onClick={() => console.log('Logo clicked!')} />
```

## Custom Size Feature

The custom size feature allows you to specify exact dimensions for the logo while maintaining its aspect ratio:

### Both Width and Height
When both `width` and `height` are specified, the logo uses the exact dimensions provided.

### Single Dimension
When only `width` or `height` is specified, the component automatically calculates the other dimension to maintain the logo's original aspect ratio (161:20 = 8.05) using the AspectRatio component.

### Supported Units
- **Numbers**: Treated as pixels (e.g., `width={200}` = 200px)
- **Strings**: Any valid CSS unit (e.g., `width="15rem"`, `height="3em"`, `width="50%"`)

## AspectRatio Component Integration

The Logo component internally uses the **AspectRatio component** to maintain proper proportions when using custom sizes with only one dimension specified:

- **When both `width` and `height` are provided**: The logo uses exact dimensions without AspectRatio
- **When only `width` is provided**: AspectRatio automatically calculates the height based on the logo's original ratio (8.05:1)
- **When only `height` is provided**: AspectRatio automatically calculates the width based on the logo's original ratio (8.05:1)
- **When neither is provided with custom size**: The logo falls back to auto sizing

This ensures the logo always maintains its visual integrity and proportions, regardless of how you specify the custom dimensions.

### RTL Best Practices

1. **Always use ThemeProvider** for proper RTL support and layout direction
2. **Provide Arabic ARIA labels** when in RTL mode for accessibility
3. **Test clickable functionality** to ensure proper interaction in RTL
4. **Consider logo placement** - typically starts from the reading direction
5. **Test custom sizing** with different units in RTL layouts
6. **Maintain aspect ratio** regardless of text direction
7. **Use semantic HTML** with proper `dir` attributes when needed
